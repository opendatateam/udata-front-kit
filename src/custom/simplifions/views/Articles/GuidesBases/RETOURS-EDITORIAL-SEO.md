# Retours éditoriaux et SEO — Guide de base pour les petites collectivités

## Retours éditoriaux

### Structure et ordre des sections

L'ordre actuel crée une friction de lecture : le lecteur arrive sur une liste de "2 vérifications à faire" avant même de comprendre le contexte. La section "De quoi parle-t-on ?" est trop tardive — elle devrait logiquement précéder les actions concrètes.

**Ordre suggéré :**
1. De quoi parle-t-on ? *(contexte)*
2. 2 vérifications à faire *(point d'entrée rapide)*
3. Les sites de consultation
4. Les logiciels éditeurs
5. Intégration directe
6. Checklist de départ

---

### Section "Les sites de consultation"

**Problème majeur** : les trois cartes (CAF pro, Annuaire des entreprises, Annuaire associations) n'ont aucun lien. Le lecteur ne peut pas aller sur ces sites. C'est un trou d'usage critique — une carte sans lien ne sert ni l'utilisateur ni le SEO.

---

### Section "De quoi parle-t-on ?"

Le titre interne est très long : *"Simplifier une démarche en récupérant la donnée à la source : de quoi parle-t-on ?"* — à tronquer, le sens est dans la seconde partie seulement.

La phrase *"Vous utilisez déjà sûrement des données récupérées à la source via vos différents logiciels éditeurs, redécouvrez lesquelles en recherchant votre éditeur ici."* (section "2 vérifications") est syntaxiquement lourde.

---

### Section "Intégration directe"

Trop légère : deux paragraphes génériques, aucun lien ou CTA. La section promet une suite mais ne la donne pas. À compléter ou à supprimer si le contenu n'est pas encore prêt.

---

### Checklist

Deux items sont trop abstraits pour une petite collectivité :
- *"Le besoin métier est clairement identifié"* — vrai pour tout projet, pas spécifique
- *"Le premier périmètre reste limité et réaliste"* — idem

Ils gagneraient à être plus concrets, par ex. : *"Vous avez identifié au moins une démarche où vous demandez un document déjà connu d'une autre administration."*

---

### Ton

Globalement accessible et cohérent. Un point : *"Simplifions.data.gouv.fr référence les logiciels métiers..."* — le lecteur est déjà sur le site, mieux vaut écrire *"Ce site"* ou simplement *"Simplifions"*.

---

## Retours SEO

### Titre (`title`)

`Guide de base pour les petites collectivités` — trop générique. Il n'inclut aucun mot-clé sur le sujet traité. Suggestions :

- *"Comment simplifier les démarches administratives dans une petite collectivité"*
- *"Guide pratique : simplification des démarches pour les petites communes, sans développement"*

---

### Description (balise meta)

`Récupérer les données de vos usagers pour éviter de leur redemander lors de leurs démarches administratives, c'est aussi possible pour les petites collectivités.`

- L'expression *"récupérer les données de vos usagers"* peut être lue comme une collecte de données personnelles — à reformuler pour éviter toute ambiguïté RGPD.
- Pas de verbe d'action, pas de bénéfice concret mis en avant.
- L'USP forte de cet article (*"sans développement informatique"*) n'est pas mentionnée.

**Suggestion :** *"Découvrez comment simplifier vos démarches administratives sans développement : portails publics gratuits, logiciels éditeurs déjà raccordés. Un guide pour les petites collectivités."*

---

### Labels de sections (utilisés comme titres H2/H3)

| Actuel | Plus SEO-friendly |
|---|---|
| `De quoi parle-t-on ?` | `Le principe Dites-le nous une fois expliqué` |
| `Les sites de consultation` | `Portails publics gratuits : consulter les données sans développement` |
| `Les logiciels éditeurs` | `Vos logiciels métiers : les données déjà disponibles` |
| `Intégration directe` | `Aller plus loin : intégration directe par API` |

---

### Maillage interne

La section "De quoi parle-t-on ?" cite le principe *Dites-le nous une fois* sans jamais le lier à une ressource externe (service-public.fr, la DITP...). C'est un terme recherché qui mériterait un lien sortant pour appuyer la crédibilité.

---

### Tags (`articleTags`)

`Petites communes` et `Maires` ciblent bien l'audience. Mais si ces tags sont exploitables en filtres ou en balisage, ajouter `Dites-le nous une fois` ou `Simplification administrative` renforcerait la cohérence thématique du site.
